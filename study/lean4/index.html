research bridge

          
从 Lean 入门到 AI 解数学难题

          
一周后你不需要会很多高等数学，但应该能进入“形式化证明 + 机器学习”的工作流。

        

        

          

            
1. 形式化题目

            
把自然语言题目拆成对象、假设、目标。Lean 的价值是让“题目是否表达正确”变成可检查对象。

            
theorem target (a b : Nat) (h : a = b) : b = a := by
  exact h.symm

          

          

            
2. 构造证明搜索空间

            
AI 不是直接“想出证明”，而是生成 tactic、调用检索、检查状态，再循环修正。

            
state -> retrieve -> propose tactic -> check -> new state

          

          

            
3. 使用 Mathlib

            
Mathlib 是 Lean 数学库。科研中大量能力来自找定理、改写定理、组合已有 lemma。

            
#check Nat.add_comm
#check mul_assoc
#check Set.Subset.antisymm

          

          

            
4. 建立实验指标

            
不要只看生成文本质量。看可编译率、证明完成率、搜索步数、定理检索命中率与错误类型分布。

            
pass@k, compile rate, tactic accuracy, proof length
